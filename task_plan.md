# 升级计划：Student Hub → 企业级后台管理系统

## 目标
功能完善 + UI优化，简洁、专业、统一、真实可用。

## 不改动的
- 数据库 schema（已足够好）
- JWT 认证逻辑
- CSV 导入/导出逻辑
- Docker 配置
- 现有 API 端点签名（不破坏兼容性）

## 改动清单

### Phase 1: 后端增强
1. `main.py` — 添加请求追踪中间件（X-Request-ID）
2. `routers/stats_router.py` — 新增 `/stats/dashboard` 端点，返回仪表盘汇总+最近学生

### Phase 2: 前端 UI 重构（index.html）
1. **全局布局优化**
   - 侧边栏加工具提示（title 属性）
   - 添加顶部导航栏（含面包屑 + 快捷操作）
   - 统一页面头部样式

2. **仪表盘页**
   - 统计卡片增加趋势指示
   - 添加「最近添加的学生」表格（Top 5）
   - 优化空状态

3. **学生管理页**
   - 表格操作列优化（图标按钮 + tooltip）
   - 添加「批量操作」下拉菜单
   - 优化分页组件
   - 搜索框防抖
   - 筛选条件保存到 URL

4. **数据分析页**
   - 添加汇总卡片行
   - 图表布局优化

5. **设置页**
   - 添加系统信息展示
   - 显示 API 版本、数据库状态

6. **通用改进**
   - Toast 支持 info/warning 类型
   - 加载状态骨架屏
   - 网络错误重试提示
   - 键盘快捷键帮助面板

### Phase 3: 验证
1. 后端启动测试
2. 登录/注册流程
3. 学生 CRUD
4. CSV 导入/导出
5. 统计页
6. user_id 数据隔离
